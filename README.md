# supply_chain_project
Developed a Supply Chain Management database using SQL to analyze customer behavior, supplier performance, and product demand through complex queries, joins, and aggregations.
Supply Chain Management SQL Project
📌 Overview
This project is a Supply Chain Management system implemented using SQL. It demonstrates how databases can be used to manage and analyze supply chain operations such as products, suppliers, customers, and orders.

The project focuses on:
1 Data organization using relational tables
2 Writing efficient SQL queries
3 Performing business analysis using aggregation and joins


🗂️ Project Structure
📁 Supply-Chain-SQL-Project
│── DATASET.sql                # Database schema + sample data
│── questions and solution.sql # SQL queries and solutions
│── README.md                  # Project documentation


🛠️ Technologies Used
1 SQL (MySQL / PostgreSQL compatible)
2 Relational Database Concepts


🧱 Database Schema
The project includes multiple tables such as:
1 Customer – Stores customer details
2 Supplier – Stores supplier information
3 Product – Contains product details
4 Orders – Tracks customer orders
5 OrderItem – Links products with orders

📊 Key Features / Queries
🔹 Data Retrieval
Fetch all products and suppliers
List customer orders
🔹 Aggregation
Total sales per product
Revenue generated by suppliers
🔹 Joins
Combine customer, order, and product data
Analyze relationships between suppliers and products
🔹 Advanced Queries
Top 3 suppliers based on revenue
High-demand products
Customer order frequency analysis

🚀 How to Run the Project
1 Open your SQL environment (MySQL / PostgreSQL / Workbench)
2 Import the dataset:  SOURCE DATASET.sql;

3 Run queries: SOURCE "questions and solution.sql";

📈 Example Insights
1 Identify best-performing suppliers
2 Track customer purchasing behavior
3 Detect high-demand products
4 Analyze sales trends



🎯 Learning Outcomes
Through this project, you will learn:
1 SQL Joins (INNER, LEFT, etc.)
2 GROUP BY and HAVING
3 Aggregate functions (SUM, COUNT, AVG)
4 Writing real-world business queries
